1. ОБЩИЕ ПРИНЦИПЫ (ВЫСШИЙ ПРИОРИТЕТ)

- Правила в этом документе имеют приоритет над интерпретацией модели
- Если правило применимо — оно ОБЯЗАТЕЛЬНО
- Если правила конфликтуют — применяется более конкретное
- Классификация сообщения ≠ ответ клиенту
- Этот документ используется только для классификации

==================================================
2. ПРАВИЛА INTENT

--------------------------------------------------
2.1 order_status (статус заказа)

Используется, если клиент:
- спрашивает о статусе заказа
- уточняет, был ли заказ отправлен
- интересуется сроками без жалобы

--------------------------------------------------
2.2 delivery_issue (проблема с доставкой)

Используется ТОЛЬКО если:
- заказ не доставлен
- есть задержка доставки
- проблема с курьером или адресом

--------------------------------------------------
2.3 delivery_update (изменение данных доставки)

Используется ТОЛЬКО если:
- клиент просит изменить получателя
- клиент просит изменить адрес доставки
- клиент просит изменить телефон или контактные данные
- изменение НЕ связано с жалобой или проблемой доставки

Примеры:
- "Нужно поменять получателя"
- "Измените номер телефона для доставки"
- "Исправьте адрес доставки"

--------------------------------------------------
2.4 payment_issue (проблема с оплатой)

Используется, если есть проблема с оплатой или деньгами.

ЖЁСТКО:
- payment_issue → urgency не может быть low

--------------------------------------------------
2.5 cancel_order (отмена заказа)

Используется, если клиент явно хочет отменить заказ.

--------------------------------------------------
2.6 return_refund (возврат или возврат денег)

Используется, если клиент:
- хочет вернуть товар
- просит возврат денег

--------------------------------------------------
2.7 product_composition_question (комплектация товара)

Используется ТОЛЬКО если вопрос:
- о комплектации товара
- о составе набора
- о том, что входит или не входит в комплект
- о различиях комплектаций

--------------------------------------------------
2.8 product_availability_question (наличие и покупка товара)

Используется ТОЛЬКО если вопрос:
- о возможности покупки товара
- о наличии товара
- о том, продаётся ли товар или можно ли его заказать

--------------------------------------------------
2.9 product_question (вопрос о товаре)

Используется, если:
- вопрос о характеристиках
- вопрос о совместимости
- вопрос об использовании
- вопрос об оригинальности

ВАЖНО:
- Вопросы о комплектации и наличии ≠ product_question

--------------------------------------------------
2.10 technical_issue (техническая проблема)

Используется, если:
- товар не работает
- произошла поломка
- требуется ремонт

--------------------------------------------------
2.11 complaint (жалоба)

Используется ТОЛЬКО если:
- выражено недовольство
- НЕТ конкретного запроса или действия

--------------------------------------------------
2.12 customer_feedback (обратная связь)

Используется ТОЛЬКО если:
- благодарность
- подтверждение получения заказа
- информация без запроса

ЖЁСТКО:
- customer_feedback → urgency = low

--------------------------------------------------
2.13 other (другое)

Используется, если intent неочевиден или недостаточно контекста.

==================================================
3. ПРАВИЛА РАЗРЕШЕНИЯ КОНФЛИКТОВ

- delivery_update имеет приоритет над delivery_issue,
  если нет жалобы или проблемы
- product_composition_question имеет приоритет над product_question
- product_availability_question имеет приоритет над product_question
- Если есть конкретная проблема → intent ≠ complaint
- payment_issue имеет приоритет над return_refund
- delivery_issue имеет приоритет над complaint
- EDGE + неопределённость → intent = other

==================================================
4. ПРАВИЛА URGENCY

high:
- деньги с негативом
- срочный возврат денег
- сильный негатив
- угрозы
- "мошенники"

medium:
- есть проблема без эскалации

low:
- информационный запрос
- customer_feedback

==================================================
5. EDGE ПРАВИЛА

edge = true, если:
- 1–5 слов
- нет контекста
- нет глаголов

EDGE снижает confidence.

==================================================
6. CONFIDENCE

- edge = true → confidence ≤ 0.6
- intent = other → confidence ≤ 0.5
- customer_feedback → confidence ≥ 0.8

==================================================
7. ФИНАЛЬНЫЙ FALLBACK

Если есть сомнения:
- intent = other
- urgency = medium
- confidence ≤ 0.5
